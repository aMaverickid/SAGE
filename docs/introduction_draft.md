Introduction Draft

[Opening / Task]
Sparse autoencoders provide a promising interface for interpreting language model internals, but their usefulness depends on whether each learned feature can be assigned a faithful natural-language explanation. Recent automated interpretability systems address this problem with agentic workflows: they inspect high-activation exemplars, propose candidate hypotheses, generate active tests, observe feature activations, and refine the explanation until a final description is produced. This workflow is attractive because it can gather targeted evidence, but it also makes feature interpretation expensive: each additional test and refinement step consumes LLM calls, activation queries, and wall-clock time.

[Challenge]
A central failure mode of agentic feature interpretation is refinement spin, where additional input-side tests stop changing the agent's evidence but the workflow continues to refine. In this regime, the agent may repeatedly rewrite hypotheses without learning new information about the feature. Output-centric signals such as vocabulary projections or steering effects appear to offer a natural escape, because they expose how a feature affects model outputs rather than only which inputs activate it. However, our experiments show that output-centric evidence is not automatically beneficial: its effect depends both on the signal type and on where it enters the agentic workflow. In particular, upfront output priors do not reliably improve explanations, while steering evidence is more useful as a targeted intervention once input-side refinement has stopped producing new evidence. This shifts the question from whether output-centric signals help in general to how an interpretation agent should decide when to use them.

[Method / Insight]
We propose SHES-OCRS, a stagnation-based early-termination mechanism for agentic SAE feature interpretation, to answer this design question through a stagnation-based control policy. The key idea is to decide when to stop refining from the behavior of the agent's own tests. SHES maintains a Hypothesis Evidence Score for each active hypothesis, computed from the observed activation margin of each test relative to a per-feature threshold. When an individual hypothesis shows little score movement over a small pre-registered window, SHES treats that hypothesis's input-side testing loop as stalled and triggers OCRS-style commitment while other active hypotheses continue their own lifecycles.

[Technical Advantage]
SHES-OCRS uses commitment as the primary cost-control mechanism and steering evidence as a secondary causal probe. Once stagnation is detected, the agent optionally collects one steering-based output signal, injects it into the hypothesis update prompt, and forces a terminal decision rather than returning to open-ended test design. If the steering evidence is incoherent, the method falls back to the normal workflow instead of forcing an unreliable conclusion. Before final synthesis, a conservative pruning step removes dominated hypotheses only when they have low evidence and do not cover high-activation exemplars, preserving minority facets.

[Evidence / Evaluation]
We evaluate SHES-OCRS on SAE features from Gemma-2-2B using both full descriptions and short labels. This dual-surface evaluation is important because full descriptions measure input-side activation faithfulness, while labels emphasize compressed output-side discriminability. Our current OCRS-only prototype already suggests the mechanism behind the method: forced commitment nearly matches the SAGE baseline under full-description evaluation at lower cost, while steering evidence improves label-level output discriminability. In the final SHES-OCRS evaluation, we will report input metrics, output-side steering-judge metrics, their combined score, predictive accuracy on held-out tokens, LLM calls, and per-feature cost.

[Contributions]
Our contributions are threefold. First, we introduce SHES, a low-cost evidence scoring rule that detects refinement stagnation from observable activation tests without additional LLM calls. Second, we propose SHES-OCRS, which replaces a set of heuristic early-exit rules with a single stagnation trigger followed by optional steering evidence and forced commitment. Third, we provide a dual-metric evaluation showing when commitment, steering evidence, and pruning contribute to the cost-quality tradeoff of automated SAE feature interpretation.
